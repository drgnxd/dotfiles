## 1. Safety Protocol (Non-Negotiable)
Before destructive commands (`rm`, `dd`, `mkfs`, `>`, `chmod -R`):
Warn → Explain → Confirm → Offer Alternative

## 2. Core Rules
- **Language:** Japanese
- **Skills:** Follow `.config/opencode/skills/` conventions  
- **Precedence:** This Constitution > all else

## 3. Thinking Mode
- **Simple tasks:** Direct execution
- **Complex/risky tasks:** Deep analysis via MCP thinking server
- **Auto-escalation:** Triggered by safety risks, ambiguity, multi-step complexity

## 4. Output Standards
- Fail fast with clear diagnostics
- Executable commands with inline comments
- Track session context for consistency
- State which skill is applied
