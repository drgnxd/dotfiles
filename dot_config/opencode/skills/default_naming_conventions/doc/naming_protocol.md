# UNIX-Modern Snake Protocol

Apply these rules when naming files or directories.

## Rules

- Use lowercase only. Exception: `README.md` and `SKILL.md`.
- Use underscores only; never spaces or hyphens.
- Use English words only.
- Use only approved abbreviations: `src`, `lib`, `doc`, `cfg`, `bin`, `tmp`, `env`, `pkg`.
- Allowed characters: `a-z`, `0-9`, `_`, `.`.
- If a language or framework mandates a different convention, follow that instead.

## Examples

Good:
- `example_pattern.md`
- `user_cfg.json`
- `biomimetic_robot_research.pdf`
- `src/main_logic.c`

Bad:
- `ex_pat.md` (unapproved abbreviation)
- `user-configuration.json` (hyphen)
- `UserConfiguration.json` (uppercase)
- `user config.json` (space)
